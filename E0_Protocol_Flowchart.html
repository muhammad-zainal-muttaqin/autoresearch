PHASE 1A Pipeline Decision ~2-3 days
One-Stage vs Two-Stage
2-3 representative architectures × both pipelines × 2 seeds
One-stage: YOLO 4-class detection+classification
Two-stage: Binary FFB detector → crop → classifier (EfficientNet-B0 / ResNet-18 / YOLO-cls)
Compare on dual-gate: mAP + M2/M3 + M3/M4 confusion
Prior evidence from related dataset: two-stage outperformed one-stage on maturity classification
⊳ End-of-phase: Lock pipeline. >5% confusion improvement → two-stage. <2% → one-stage. 2-5% → judgment call. If results within noise, run extra seeds.

11 Architectures × 2 Seeds — within winning pipeline
If 1-stage: sweep as 4-class detectors. If 2-stage: sweep as binary detectors + classifier variants.

PHASE 3 Final Validation ~3 days
Retrain
Train+Val Combined
All labeled data, frozen hyperparameters from Phase 2. Epochs 100→150, no early stopping.
Deploy Check
TFLite Export + Quantization
Export, verify <100MB. Optional INT8 quantization (<2% mAP drop). Tablet test if hardware available.
Calibrate
Confidence Threshold Optimization
Sweep [0.1 – 0.5] on val split. Optimize for dual-gate: mAP@0.5 + M2/M3 confusion. Track M4 recall. Free — no retraining.
Analyze
Error Stratification
Worst 20 images → categorize: small obj missed, M2/M3 confusion, occlusion, false positive, lighting, label error
⊳ End-of-phase: Comprehensive error analysis. Categorize failure modes → root cause → maps to E1–E6 priorities. Final decision document for PI review.
